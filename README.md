# Тестовое задание: Polsa Outreach

Этот репозиторий содержит решение тестового задания, разделенное на три логические части.

## Структура решения

### 1. [Проверка email-доменов](./Email%20Domain%20Validator/README.md)

Скрипт для валидации списка email-адресов через проверку MX-записей.

* **Папка:** `Email Domain Validator`
* **Функционал:** Проверяет формат и наличие MX-записей, выводит статус в консоль.
* [Перейти к инструкции запуска →](./Email%20Domain%20Validator/README.md)

### 2. [Мини-интеграция с Telegram](./Telegram%20File%20Sender/README.md)

Скрипт для чтения текста из файла и отправки его в Telegram-чат.

* **Папка:** `Telegram File Sender`
* **Функционал:** Читает `.txt`, использует переменные окружения для конфиденциальности, отправляет сообщение ботом.
* [Перейти к инструкции запуска →](./Telegram%20File%20Sender/README.md)

### 3. [Архитектурная задача](./Outreach%20Fabric%20Architecture/README.md)

Описание архитектуры для рассылки (1200 адресов) с упором на отказоустойчивость и ротацию.

* **Папка:** `Outreach Fabric Architecture`
* **Содержание:** Описание инфраструктуры, стека, мониторинга и оценка стоимости.
* [Читать описание архитектуры →](./Outreach%20Fabric%20Architecture/README.md)

---

## Общая информация

В корне репозитория находятся конфигурационные файлы для управления зависимостями и версиями Python:

* `pyproject.toml`
* `mise.toml` (для пользователей mise)

Для быстрого старта рекомендуется создать виртуальное окружение в корне или следовать инструкциям внутри каждой папки задачи.
